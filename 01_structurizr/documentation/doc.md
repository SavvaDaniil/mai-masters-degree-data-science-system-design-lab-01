

## Введение
Электронной почтой можно будет пользоваться через веб-браузер, мобильное прилождения на смартфоны и дэсктопный вариант из пакета Microsoft Office. Последний вариант имеет смысл больше, если будут добавлены какие-то новые функциональные возможности по безопасности, приватности и шифрования отправки писем, так как отправка писем через браузер, кроме снижения скорости при большом объеме писем, минусов не несет и не требует установки.


## Технологии
Предлагаю использовать для frontend Angular, чтобы статическая типизация облегчало поддержку и понимание кода, для мобильных приложений использовать нативные языки программирования Java для Android и Swift для IOS, чтобы повысить шанс конфиденциальности пользователей. Для дэсктопного варианта использовать C++, так как в этом случае играет роль повышенная безопасность и шифрование содержаний писем.

#

Для backend выбор пал на язык программирования Java фрэймворк Spring, так как данная почтовая система не требовательна сильно к скорости работы в миллисекундах, а больше требовательна к надежности, отказооустойчивости, повышении читаемости кода и надежности безопасности и шифрования контента писем и петей их передачи.
